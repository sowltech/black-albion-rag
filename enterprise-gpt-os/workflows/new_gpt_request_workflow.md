# New GPT Request Workflow

## Purpose

Create a controlled intake process for new GPT or agent requests.

## Workflow Steps

1. Requester submits business need.
2. Department lead confirms the use case is valid.
3. GPT owner is assigned.
4. Data sources are listed.
5. Risk level is assigned using `governance/risk_matrix.md`.
6. Permission level is assigned.
7. Build template is completed.
8. Approval matrix is checked.
9. Test cases are written.
10. Pilot scope is defined.
11. Audit requirements are confirmed.
12. Rollout decision is recorded.
